22 A note on the publication of the source code documentation
23 ----------------------------------------------------------
24 
25 The documentation is automatically generated from the source code by
26 [Doxygen][]. A special branch of the Git repository, [`gh-pages`][gh-pages],
27 was created that holds the [GitHub project pages][github-pages]. To make use
28 of the project page facility provided by GitHub, the root directory of a
29 repository needs to be [erased][gh-erase] first before content is added.
30 Since this would remove the source code from which the documentation is
31 generated, deleting all files in the root directory while on the `gh-pages`
32 branch was not feasible.
33 
34 The solution was to pull down the `gh-pages` branch as a submodule into an
35 empty `gh-pages\` subdirectory:
36 
37 ~~~~{.sh}
38 git submodule add -b gh-pages git@github.com:bovender/LinkTitles.git gh-pages
39 ~~~~
40 
41 This command tells git-submodule to only pull down the `gh-pages` branch as
42 a submodule -- we don't need to pull down the master branch into the same
43 repository again.
